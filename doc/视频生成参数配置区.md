    # 视频生成参数配置区

    本文档说明截图所示**智能画布 Composer 底部参数区**（以及普通画布「视频生成节点」内嵌表单）的实现：UI 结构、状态存储、渲染与事件流程、参数如何进入 `/api/canvas-video` 请求。

    截图对应场景：**引擎 = 火山引擎**、`apiKind = video`、模型如 `doubao-seedance-1-0-lite-i2v-...`。

    ## 五、事件绑定：`bindDynamicParams`

    每次 `renderDynamicParams` 后全量重绑：

    | 选择器 | 行为 |
    |--------|------|
    | `.smart-pill` | 点击 `pinned` 弹层（点击外关闭由 `closeAllSmartPopovers` 处理） |
    | `[data-smart-param]` | `setDynamicSetting(key, value)` |
    | `[data-param]` | 数字输入（如自定义时长） |
    | `[data-toggle-param]` | `settings[key] = !settings[key]` + 互斥规范化 |
    | `[data-trusted-source]` | 切换 `videoTrustedSource` 并可触发上传 |

    核心写值：

    ```3466:3520:static/js/smart-canvas.js
    function setDynamicSetting(key, value){
        settings[key] = numericKeys.has(key) ? Number(value) : value;
        if(key === 'videoProvider') settings.videoModel = '';
        if(key === 'videoMultimodal' && settings.videoMultimodal) settings.videoUseFrameRoles = false;
        normalizeSmartVideoModeSettings(settings, key === 'videoUseFrameRoles');
        persistActiveSmartSettings();
        if(layoutKeys.has(key)) renderDynamicParams();
        scheduleSave();
    }
    ```

    弹层交互：`.smart-control:hover` / `.pinned` / `.interacting` 控制 `.smart-popover` 显隐（CSS `smart-canvas.css` ~857–860）。

    ## 十一、常见问题

    **Q：换了节点，参数为什么变了？**  
    A: 每个节点有自己的 `runSettings`；切换节点时 `smartSettingsForNode` 加载该节点快照。无快照则用 localStorage 里该模式（如 `volcengine:video`）的最近参数。

    **Q：为什么 MS / Comfy 没有视频参数区？**  
    A: 视频参数仅在 `engine` 为 `api` 或 `volcengine` 且 `apiKind === 'video'` 时渲染。其他引擎走图片或工作流参数。

    **Q：改参数会立刻保存吗？**  
    A: 会。`setDynamicSetting` / toggle 均调用 `scheduleSave()`，并 `persistActiveSmartSettings()` 写入当前节点。

    **Q：「高清」和「提升到 1080p」是同一个吗？**  
    A: 智能画布 i18n 为「高清」（`videoEnableUpsample` → `enable_upsample`）；普通画布文案为「提升到 1080p」，后端字段相同。
