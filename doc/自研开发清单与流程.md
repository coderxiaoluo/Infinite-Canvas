# 自研开发清单与流程

本文档面向：**参考 Infinite Canvas 产品形态，自研一套「无限画布 + 云端 AI API」工作台**。

> **前提（按当前团队共识）**
> - **不需要 ComfyUI / 本地 GPU**
> - 算力全部走 **云端 API**（APIMart、OpenAI 兼容、ModelScope、火山引擎等）
> - **现阶段**：公司内部使用
> - **远期**：可能对外商用 → 须独立自研或取得原作者授权（见文末合规说明）

> **相关文档**
> - 现有项目架构：[`技术栈.md`](./技术栈.md)
> - 直接部署现有项目：[`内部部署.md`](./内部部署.md)
> - 内部优化项：[`项目优化.md`](./项目优化.md)

## 一、先选路径：两条线可以并行

| 路径 | 适用 | 周期 | 说明 |
|------|------|------|------|
| **A. 直接用现有项目** | 马上给团队用 | 1～2 周 | 解压 → 配 API → 内网上线，见 `内部部署.md` |
| **B. 自研新项目** | 长期可控、未来可商用 | 8～16 周（MVP） | 本文档主体；**不复制 Infinite Canvas 源码** |

**推荐组合**：A 先跑起来验证需求，B 按模块逐步替换，避免业务空窗。

```
第 1～2 周    路径 A：现有项目内网试点
第 2 周起     路径 B：并行启动自研（Provider → 任务 → 画布）
第 8～12 周   自研 MVP 覆盖主流程，内部切换
远期对外      确认代码独立 或 取得商用授权
```

## 二、功能范围（无 ComfyUI 版）

### 2.1 MVP 必做（第一期）

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| **Provider 管理** | 增删改 Provider、填 Key、验证连接、拉取模型列表 | P0 |
| **API 生图** | 文生图 / 图生图，异步任务（创建 → 轮询 → 结果） | P0 |
| **无限画布** | 节点拖拽、连线、缩放平移、保存/加载 | P0 |
| **核心节点** | Prompt、Image、API 生成、Output | P0 |
| **文件上传** | 参考图上传、生成结果本地存储 | P0 |
| **画布持久化** | 画布 CRUD、JSON 存盘或数据库 | P0 |

### 2.2 第二期（团队用起来后再加）

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| **视频生成** | 文生视频 / 图生视频（火山、APIMart 等） | P1 |
| **LLM 节点** | 画布内 LLM、多模态（图+文） | P1 |
| **ModelScope** | 免费/低成本试玩通道 | P1 |
| **循环节点** | 同 Prompt 批量出图 | P1 |
| **素材库** | 分类、上传、从画布入库 | P1 |
| **GPT 对话页** | 独立多轮对话（可选） | P2 |

### 2.3 明确不做（当前阶段）

- ❌ 本地 ComfyUI 对接
- ❌ LTX Director / ComfyUI 工作流编辑器
- ❌ 即梦 CLI（除非团队明确需要）
- ❌ RunningHub 工作流（无 ComfyUI 时可整模块跳过）
- ❌ 商用计费 / 多租户（内部阶段不做，对外前再规划）

### 2.4 画布节点对照（自研最小集）

| 节点 | 作用 | 对应 API |
|------|------|----------|
| `prompt` | 输入提示词 | — |
| `image` | 参考图 / 结果图卡片 | 上传接口 |
| `generator` | 云端 API 生图 | `POST /api/canvas-image-tasks` |
| `video` | 云端视频生成 | `POST /api/canvas-video` |
| `llm` | 云端大模型 | `POST /api/canvas-llm` |
| `loop` | 批量重复执行 | 前端编排 + 多次调用生图 API |
| `output` | 汇总展示、导出 | — |

## 三、推荐技术栈

### 3.1 自研项目（路径 B）

| 层级 | 选型 | 说明 |
|------|------|------|
| 后端 | Python 3.10+ / **FastAPI** | 与现有生态一致，AI 库丰富 |
| HTTP 客户端 | **httpx**（异步） | 生图/视频长超时 |
| 任务队列 | 第一期：**JSON 文件持久化**；第二期：Redis + Worker | 防重启丢任务 |
| 存储 | 第一期：JSON + 本地文件夹；第二期：PostgreSQL + OSS | 内部 MVP 可先用 JSON |
| 前端 | **React + TypeScript + Vite** | 画布组件化、团队协作友好 |
| 画布渲染 | 自研 Canvas/SVG，或 **React Flow** / **Konva** | 节点连线成熟方案 |
| 样式 | Tailwind CSS（构建版，非 CDN） | |
| 实时 | WebSocket | 任务完成广播、在线状态（可选） |

### 3.2 目录结构建议

```text
your-ai-canvas/
├── backend/
│   ├── main.py                 # 入口
│   ├── routes/
│   │   ├── providers.py        # Provider CRUD、测试连接
│   │   ├── canvas.py           # 画布 CRUD
│   │   ├── tasks.py            # 生图/视频/LLM 异步任务
│   │   ├── upload.py           # 文件上传
│   │   └── assets.py           # 素材库
│   ├── services/
│   │   ├── task_manager.py     # 任务创建、轮询、持久化
│   │   └── media_storage.py    # 本地/OSS 存图
│   ├── providers/              # 各 AI 平台适配（核心）
│   │   ├── base.py             # 抽象接口
│   │   ├── openai_compat.py    # OpenAI 兼容协议
│   │   ├── apimart.py          # APIMart 异步协议
│   │   ├── modelscope.py
│   │   └── volcengine.py       # 视频
│   ├── models/                 # Pydantic 请求/响应模型
│   └── config/                 # 配置加载、Key 管理
├── frontend/
│   ├── src/
│   │   ├── pages/
│   │   │   ├── CanvasPage.tsx      # 无限画布
│   │   │   ├── ApiSettingsPage.tsx # API 设置
│   │   │   └── AssetLibraryPage.tsx
│   │   ├── components/canvas/      # 节点、连线、工具栏
│   │   ├── hooks/                  # 任务轮询、WebSocket
│   │   └── api/                    # 后端 API 封装
│   └── package.json
├── data/                       # 画布 JSON、Provider 配置
├── assets/                     # 上传与生成文件
└── docker-compose.yml          # 可选，后期容器化
```

## 四、后端开发清单

### 4.1 Provider 层（Week 1～2）

- [ ] 定义 `BaseProvider` 接口：`test_connection()`、`fetch_models()`、`generate_image()`、`generate_video()`、`chat()`
- [ ] 实现 **OpenAI 兼容** 协议（`/v1/images/generations`、`/v1/chat/completions`）
- [ ] 实现 **APIMart 异步** 协议（提交任务 + 轮询，参考官方文档，非抄源码）
- [ ] Provider 配置读写（`api_providers.json` 或 DB）
- [ ] API Key 存 `.env` / 环境变量，接口返回时脱敏
- [ ] `POST /api/providers/test-connection`
- [ ] `GET/PUT /api/providers`
- [ ] `POST /api/providers/fetch-models`

**验收**：在 API 设置页填 Key → 验证通过 → 拉取模型列表 → 保存成功。

### 4.2 生图任务（Week 2～3）

- [ ] `POST /api/canvas-image-tasks` → 返回 `task_id`
- [ ] `GET /api/canvas-image-tasks/{task_id}` → `queued | running | completed | failed`
- [ ] 任务状态 **持久化到磁盘**（`data/tasks/{id}.json`），重启可恢复
- [ ] 后台异步执行：httpx 调上游 → 下载图片 → 存 `assets/output/`
- [ ] 支持参考图：`reference_images[]`
- [ ] 超时与错误信息友好返回

**验收**：curl 创建任务 → 轮询拿到本地图片 URL；重启服务后旧任务仍可查询。

### 4.3 文件与画布（Week 3～4）

- [ ] `POST /api/ai/upload`  multipart 上传
- [ ] `GET/POST/PUT/DELETE /api/canvases` 画布 CRUD
- [ ] 画布 JSON 结构：`nodes[]`、`edges[]`、`viewport`、`meta`
- [ ] 保存冲突检测（可选，多人时用 `updated_at` + 409）
- [ ] 静态文件服务：`/assets/*`

**验收**：创建画布 → 添加节点 → 保存 → 刷新后数据一致。

### 4.4 视频与 LLM（Week 5～6，第二期）

- [ ] `POST /api/canvas-video`（同步长请求或异步任务，与上游一致）
- [ ] `POST /api/canvas-llm`（支持 images[] 多模态）
- [ ] ModelScope 独立适配（可选）
- [ ] 火山引擎视频签名（若需要）

**验收**：Prompt + 参考图 → 视频节点 → Output 可见；LLM 节点读图回答。

### 4.5 素材库（Week 6～7，第二期）

- [ ] `GET/POST /api/asset-library/*` 分类与条目
- [ ] 从画布图片一键入库
- [ ] 缩略图生成（Pillow）

### 4.6 内部安全（与功能并行）

- [ ] CORS 限定内网域名（生产勿用 `*`）
- [ ] 简单访问 Token 或 Nginx Basic Auth 中间件
- [ ] 日志中 Key 脱敏
- [ ] `API/.env`、`data/` 不进 Git

## 六、开发流程（按周）

### Phase 0：准备（3～5 天）

1. 确认团队人数、内网服务器、API 账号（APIMart 等）
2. 新建 **独立 Git 仓库**（不要 fork Infinite Canvas）
3. 搭建 backend + frontend 空项目
4. 对照 [`技术栈.md`](./技术栈.md) 理解 API 设计，**只看接口与流程，不复制代码**
5. 用现有 Infinite Canvas 跑通一次：生图 → 保存画布，录屏给团队对齐预期

### Phase 1：后端 + 设置页（Week 1～2）

```
Provider 抽象 → OpenAI 兼容 → APIMart 异步
     ↓
API 设置页联调
     ↓
在线生图 API（同步，便于调试）
```

**里程碑 M1**：命令行 / Postman 能调通生图。

### Phase 2：画布 MVP（Week 3～5）

```
画布 CRUD → 节点渲染 → 连线 → Generator 节点调异步生图
     ↓
Prompt + Image → Generator → Output 全链路
```

**里程碑 M2**：浏览器内完成第一张「画布生图」。

### Phase 3：内部试点（Week 6～7）

```
任务持久化 → 上传/素材 → Nginx 内网部署
     ↓
3～5 人试用，收集反馈
```

**里程碑 M3**：同事通过内网 IP 日常使用。

### Phase 4：第二期能力（Week 8～12）

```
Video / LLM / Loop / 素材库
     ↓
WebSocket 广播（可选）
     ↓
备份脚本、运维文档
```

**里程碑 M4**：覆盖团队 80% 日常场景（无 ComfyUI）。

### Phase 5：对外前（时间未定）

- 评估：代码是否已独立于 Infinite Canvas
- 选项 A：联系原作者 **商用授权**
- 选项 B：补齐用户体系、计费、PostgreSQL、OSS、内容审核后 **独立上线**

## 七、API 设计参考（自研时对齐，非照搬）

以下为 **行业常见设计**，实现时以各平台官方文档为准：

| 接口 | 方法 | 用途 |
|------|------|------|
| `/api/providers` | GET/PUT | Provider 配置 |
| `/api/providers/test-connection` | POST | 验证 Key |
| `/api/canvas-image-tasks` | POST | 创建生图任务 |
| `/api/canvas-image-tasks/{id}` | GET | 查询任务 |
| `/api/canvas-video` | POST | 视频生成 |
| `/api/canvas-llm` | POST | LLM 调用 |
| `/api/canvases` | GET/POST | 画布列表/创建 |
| `/api/canvases/{id}` | GET/PUT | 读写画布 |
| `/api/ai/upload` | POST | 上传参考图 |
| `/ws/stats` | WebSocket | 任务/画布广播（可选） |

画布 JSON 最小示例：

```json
{
  "id": "canvas_001",
  "title": "产品海报",
  "viewport": { "x": 0, "y": 0, "zoom": 1 },
  "nodes": [
    { "id": "n1", "type": "prompt", "x": 100, "y": 200, "text": "a product photo" },
    { "id": "n2", "type": "generator", "x": 400, "y": 200, "apiProvider": "apimart", "model": "gpt-image-2", "inputs": ["n1"] },
    { "id": "n3", "type": "output", "x": 700, "y": 200, "images": [], "inputs": ["n2"] }
  ],
  "updated_at": "2026-06-22T10:00:00Z"
}
```

## 八、人力与周期估算

| 配置 | MVP（P0） | 含第二期（P1） |
|------|-----------|----------------|
| 1 全栈 + AI 辅助 | 10～14 周 | 14～18 周 |
| 1 后端 + 1 前端 | 6～8 周 | 10～12 周 |
| 小团队 + Cursor/Codex | 可缩短 20%～30% | 画布交互仍是最耗时部分 |

**最大风险点**：无限画布前端（节点、连线、执行引擎），建议优先验证技术方案（React Flow 等），再全面铺开。

## 九、测试清单

### 9.1 联调测试

- [ ] Provider 验证失败时错误提示正确

## 十、合规说明（必读）

| 行为 | 公司内部 | 对外商用 |
|------|----------|----------|
| 直接使用 Infinite Canvas | ✅ | ❌ 需授权 |
| 参考产品思路自研 | ✅ | ✅（代码须独立） |
| 对照源码让 AI 重写后商用 | ⚠️ 内部可 | ❌ |
| 复制 / fork 后换皮 | ⚠️ 内部可 | ❌ |

**自研原则**：

1. **新建仓库**，不 fork 原项目  
2. 参考 **API 官方文档** 和 **产品交互**，不粘贴 `main.py` / `canvas.js`  
3. 对外前做代码自查：结构、命名、注释是否与原版大量雷同  
4. 若高度依赖原版，对外前应 **联系原作者取得商用授权**（B 站：README 链接）
