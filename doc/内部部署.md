# 内部部署指南（无 GPU / 无 ComfyUI）

本文档适用于：**公司内网部署 Infinite Canvas，全部生成能力走云端 API**，不配置本地 ComfyUI、不依赖显卡。

---

## 一、架构概览

```
┌──────────────────────────────────────────────┐
│  普通 Windows / Linux 服务器（无需 GPU）      │
│  Infinite Canvas  :3000                      │
│  仅负责：网页 + 编排 + 转发 API + 存本地文件   │
└──────────────────────────────────────────────┘
         │ 内网访问（建议 Nginx + 密码）
         │
    设计师 / 运营（浏览器）
         │
         ▼ HTTPS 出网（仅服务器需能访问外网）
┌──────────────────────────────────────────────┐
│  云端 API（任选一种或多种）                   │
│  · APIMart / OpenAI 兼容中转站（推荐主用）    │
│  · ModelScope（免费额度，适合试玩/学习）      │
│  · 火山引擎（视频，如 doubao-seedance）       │
│  · RunningHub（云端 Comfy 工作流，免本地 GPU）│
└──────────────────────────────────────────────┘
```

**要点**：
- 画布、素材、历史存在服务器本地 `data/`、`assets/`
- **算力在云端**，公司机器只跑 Web 服务
- 无需安装 CUDA、ComfyUI、模型文件

---

## 二、硬件与系统要求

| 项目 | 建议 |
|------|------|
| CPU | 4 核及以上 |
| 内存 | 8 GB 及以上（16 GB 更稳，多人同时用） |
| 显卡 | **不需要** |
| 磁盘 | 50 GB+（存生成图/视频、素材库） |
| 系统 | Windows 10/11 或 Linux |
| 网络 | 服务器能访问外网 API；同事通过内网访问 `:3000` |

## 三、部署步骤

### 1. 解压并安装依赖

```text
1. 解压项目到固定目录，例如 D:\Infinite-Canvas\
2. 双击 安装依赖.bat（或 macOS 运行 mac-安装依赖.sh）
3. 确认 python\python.exe 存在，或本机已装 Python 3.10+
```

### 2. 配置 API（管理员一次性完成）

1. 双击 `run.bat` 启动，浏览器打开 `http://127.0.0.1:3000/`
2. 进入 **API 设置**
3. 新增或编辑 Provider，例如：

| 用途 | 协议 | 请求地址示例 |
|------|------|--------------|
| 主用（生图/视频/LLM） | APIMart 异步 | `https://api.apimart.ai` |
| 免费试玩 | OpenAI（ModelScope） | `https://api-inference.modelscope.cn/v1` |
| 视频（火山） | 火山引擎 | 按火山控制台填写 |

4. 填入 API Key → **验证地址** → **拉取模型** → 勾选要开放的模型 → **保存**

Key 保存在项目目录 `API/.env`，**不要提交 Git、不要发到群聊**。

### 3. 内网开放给同事

默认 `run.bat` 启动后监听 `0.0.0.0:3000`，同事可通过：

```text
http://<服务器内网IP>:3000/
```

例如 `http://192.168.1.100:3000/`。

**安全建议**（至少做一项）：
- 仅在内网使用，路由器不要映射 3000 到公网
- 前面加 Nginx，配置 Basic Auth
- 或由 IT 限制只有公司网段能访问该 IP

### 4. 开机自启（可选）

Windows：任务计划程序，登录时运行 `run.bat`  
Linux：`systemd` 单元调用 `python main.py`

## 四、团队推荐使用方式（无 ComfyUI）

### 画布节点选型

| 需求 | 使用节点 | 说明 |
|------|----------|------|
| 文生图 / 图生图 | **API 生成** | 选 APIMart 等 Provider |
| 免费试玩 | **MS 生成** | ModelScope Token |
| 文生视频 / 图生视频 | **视频生成** | 如火山 doubao-seedance |
| 文案 / 看图说话 | **LLM** | 选对话模型 |
| 批量同 prompt | **循环** | 多图批量 |
| 汇总结果 | **Output** | 查看与导出 |

**可不用的功能**（无 GPU 时）：
- 工具栏 **ComfyUI** 节点
- **LTX Director**（依赖 ComfyUI 工作流）
- 左下角 ComfyUI 设置（无需配置）

若需要 ComfyUI 类工作流、又不想本地 GPU，可改用 **RunningHub 生成（RH 生成）** 节点 + RunningHub Key（云端跑工作流）。

### 推荐工作流模板（示例）

**模板 A：产品图 → 视频**

```text
[Prompt 提示词] ──┐
[Image 产品图]  ──┼→ [视频生成] → [Output]
```

**模板 B：海报多尺寸**

```text
[Prompt] ──→ [API 生成] → [Output]
              （换 size 复制节点或 Loop）
```

模板 JSON 可存 `data/templates/`，新建画布时导入。

## 五、日常运维

### 备份（建议每周）

打包备份以下目录/文件：

```text
data/           # 画布、素材库索引、配置
assets/         # 上传与生成文件
API/.env        # Key（单独加密存放）
history.json    # 生成历史
```

### 磁盘清理

定期清理或归档：
- `output/` 旧生成物
- `data/media_previews/` 缩略图缓存

### 常见问题

| 现象 | 处理 |
|------|------|
| 拉取模型失败 | 检查 Key、账户余额、服务器能否访问外网 |
| 视频生成很慢 | 正常，云端排队；避免多人同时跑大量长视频 |
| 服务重启后任务消失 | 见 `doc/项目优化.md` 任务持久化改造 |
| 端口被占用 | 关闭旧的黑窗口或 `netstat` 查 3000 进程 |

## 六、成本与账号

- **APIMart / 中转站**：按量计费，管理员在平台控制台看账单
- **ModelScope**：有免费额度，需实名；适合内部试玩，不适合 heavy 生产
- **火山视频**：按次/按秒计费，视频节点消耗相对较高

建议：公司统一一个主账号 + 统一 Key，内部不再每人申请 Key。

---

## 七、与「有 GPU / ComfyUI」方案的区别

| 项目 | 无 GPU（本文） | 有 GPU + ComfyUI |
|------|----------------|------------------|
| 服务器 | 普通 PC 即可 | 建议 NVIDIA 16GB+ 显存 |
| 生图成本 | 云 API 按量 | 本地免费，电费+机器 |
| 定制工作流 | RunningHub 或 API | 本地 ComfyUI 任意工作流 |
| 维护难度 | 低 | 高（模型、节点、驱动） |
| 数据隐私 | 提示词/图片会发到 API 商 | 本地 Comfy 可完全离线 |

**无 GPU 时**：优先 **API 生成 + 视频生成 + LLM**；复杂工作流用 **RunningHub**，不必强求本地 ComfyUI。
