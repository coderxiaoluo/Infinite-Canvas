# 自研起步清单

面向：**参考 Infinite Canvas 产品形态，从零启动一套「无限画布 + 云端 AI API」自研项目**。

> 本文是「开工前 + 前 2 周」的行动清单。完整模块拆解与按周排期见 [`自研开发清单与流程.md`](./自研开发清单与流程.md)；架构说明见 [`技术栈.md`](./技术栈.md)。

---

## 一、开工前先想清楚（Day 0）

在写第一行代码前，团队对齐下面 5 项（填表或开会记录）：

| 项 | 你要定的答案 | 示例 |
|----|--------------|------|
| 产品形态 | 节点工作流 / 自由白板 / 两者都要 | 先做 **节点工作流**（Prompt→生图→Output） |
| 算力来源 | 仅云端 API / 含本地 ComfyUI | 第一期 **仅云端 API** |
| 使用范围 | 个人 / 公司内部 / 将来对外商用 | 先 **公司内部** |
| 团队 | 几人、前后端分工 | 1 全栈 或 1 后端 + 1 前端 |
| 周期目标 | MVP 何时可用 | 例如 **8 周内**浏览器内跑通生图 |

**默认推荐（最省风险）**

- 节点工作流 + 云端 API + 公司内部 + React Flow 画布 + FastAPI 后端
- 第一期 **不做**：ComfyUI、RunningHub、即梦 CLI、计费、多租户、实时协同

## 二、合规与参考边界（必读）

自研 **可以** 参考产品思路、交互、API 设计；**不要** 大段复制原项目代码后闭源商用。

| 行为 | 公司内部 | 将来对外商用 |
|------|----------|--------------|
| 参考节点类型、任务流程、接口设计 | ✅ | ✅ |
| 新建独立仓库，自己写代码 | ✅ | ✅（代码须足够独立） |
| 直接使用 / Fork 原 Infinite-Canvas | ✅ | ❌ 需原作者授权 |
| 复制 `main.py` / `canvas.js` 改皮 | ⚠️ 仅建议短期内部试点 | ❌ |
| 闭源打包销售 | ❌ | ❌ |

**自研原则**

1. **新建 Git 仓库**，不要 Fork 原项目当长期主库  
2. 参考 **官方 API 文档** + 原项目的 **接口形态**，实现自己写  
3. 若基于原代码二次改动并分发 → 须 **开源 + 注明原作者**（见根目录 `LICENSE`）  
4. 对外商用前 → 联系原作者 **取得授权**，或完成独立重写并做合规自查

## 四、技术选型（固定下来，避免中途换栈）

| 层级 | 选型 | 说明 |
|------|------|------|
| 后端 | **FastAPI** + **Uvicorn** | 与 AI 生态一致 |
| HTTP | **httpx**（异步） | 生图长超时 |
| 校验 | **Pydantic** | 请求/响应模型 |
| 图片 | **Pillow** | 缩略图、格式转换 |
| 上传 | **python-multipart** | 文件上传 |
| 配置 | **python-dotenv** | `.env` 读 Key |
| 前端 | **Vite + React + TypeScript** | 组件化、易协作 |
| 画布 | **React Flow** | 节点连线（勿第一期自研 DOM 画布） |
| 样式 | **Tailwind CSS**（构建版） | 非 CDN |
| 存储 MVP | **JSON 文件 + 本地目录** | 后期 PostgreSQL + OSS |
| 任务 MVP | **JSON 持久化** `data/tasks/` | 后期 Redis + Worker |

## 六、MVP 功能范围（第一期必做 P0）

做完以下项 = **最小可用产品**。

| 模块 | 功能 | 验收标准 |
|------|------|----------|
| Provider | 增删改、验 Key、拉模型 | 设置页保存后重启仍有效 |
| 生图任务 | 异步 create → poll | curl 能拿到本地图片 URL |
| 上传 | 参考图 multipart | 返回可访问 URL |
| 画布 CRUD | 创建 / 保存 / 加载 | 刷新后节点还在 |
| 画布 UI | 平移缩放、4 种节点、连线 | 可拖拽连线 |
| 执行 | Prompt→Generator→Output | 点运行，Output 出图 |

### 6.1 第一期节点（只做 4 个）

| 类型 | 作用 |
|------|------|
| `prompt` | 输入提示词 |
| `image` | 上传 / 展示参考图或结果 |
| `generator` | 调云端生图 API |
| `output` | 汇总展示生成结果 |

### 6.2 第一期 API（后端按序实现）

| 接口 | 方法 |
|------|------|
| `/health` | GET |
| `/api/providers` | GET / PUT |
| `/api/providers/test-connection` | POST |
| `/api/providers/fetch-models` | POST |
| `/api/canvas-image-tasks` | POST |
| `/api/canvas-image-tasks/{id}` | GET |
| `/api/canvases` | GET / POST |
| `/api/canvases/{id}` | GET / PUT |
| `/api/ai/upload` | POST |
| `/assets/*` | 静态文件 |

### 6.3 画布 JSON 最小结构

```json
{
  "id": "canvas_001",
  "title": "未命名画布",
  "viewport": { "x": 0, "y": 0, "zoom": 1 },
  "nodes": [
    { "id": "n1", "type": "prompt", "position": { "x": 100, "y": 200 }, "data": { "text": "a cat" } },
    { "id": "n2", "type": "generator", "position": { "x": 400, "y": 200 }, "data": { "providerId": "default", "model": "gpt-image-2" } },
    { "id": "n3", "type": "output", "position": { "x": 700, "y": 200 }, "data": { "images": [] } }
  ],
  "edges": [
    { "id": "e1", "source": "n1", "target": "n2" },
    { "id": "e2", "source": "n2", "target": "n3" }
  ],
  "updated_at": "2026-06-24T12:00:00Z"
}
```

## 十四、人力与风险

| 配置 | MVP 周期 | 最大风险 |
|------|----------|----------|
| 1 全栈 | 10～14 周 | 画布 + 执行引擎耗时 |
| 1 后端 + 1 前端 | 6～8 周 | 前后端接口对齐 |
| 小团队 + AI 辅助 | 可缩短 20～30% | 勿让 AI 对着原文件生成等价代码 |

**风险应对**

1. **画布难产** → 必须用 React Flow，第一期禁止自研 DOM 画布  
2. **Provider 过多** → MVP 只做 OpenAI 兼容一种  
3. **范围膨胀** → 严格 P0，Video/LLM 进第二期  
4. **合规** → 新仓库 + 不闭源抄代码
