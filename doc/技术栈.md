Infinite Canvas 项目技术栈与架构分析
这是一个本地运行的 AI 创作工作台：浏览器访问 http://127.0.0.1:3000/，后端统一代理多种 AI 能力（云端 API、ModelScope、ComfyUI、RunningHub、火山引擎、即梦 CLI 等），前端提供无限画布、智能画布、素材库、对话等界面。
重要授权说明：README 明确标注已申请著作权，禁止未经授权的商业用途。若要做商用产品，需取得原作者授权，或参考架构思路独立重写，不能直接封装/二次开发后闭源商用。

一、整体架构

层级	技术选型	说明
后端	Python 3.10 + FastAPI	单文件 main.py（约 1.48 万行），ASGI 服务
前端	原生 HTML/JS/CSS	无 React/Vue 构建链，多页面应用
存储	JSON + 文件夹	无数据库，画布/配置/素材均落盘
通信	REST + WebSocket	长任务异步轮询 + 实时广播
部署	便携 Python + bat 脚本	Windows 一键启动，也可系统 Python

四、项目流程（详细步骤）
4.1 安装与启动
1. 解压项目 → 可选运行 安装依赖.bat
2. 双击 run.bat / 启动服务.bat
3. python\python.exe main.py → Uvicorn :3000
4. 浏览器打开 http://127.0.0.1:3000/
启动时自动执行：静态 HTML 版本同步、素材库目录迁移、图片扩展名修复。
4.2 配置 AI 能力（三选一即可运行）
路径	步骤
付费 API	API 设置 → 填 Base URL + Key → 选协议 → 验证 → 拉取模型 → 保存
ModelScope 免费	填 Token → 验证 → 选 Z-Image 等模型，可选 LoRA
本地 ComfyUI	启动 ComfyUI:8188 → ComfyUI 设置填地址 → 导入工作流 JSON
4.3 无限画布生图流程

4.4 ComfyUI 生图流程
1. 画布选择 ComfyUI 节点 → 选工作流（如 Z-Image.json）  
2. POST /api/canvas-comfy-tasks → 后台调用 generate()  
3. 选择最优 ComfyUI 实例 → 同步 input 图片 → /prompt 提交  
4. 轮询 /history → 下载 output → 返回本地 URL
4.5 视频 / LLM 流程
● 视频：画布节点 → POST /api/canvas-video（同步长请求，内部轮询上游）  
● LLM：画布 LLM 节点 → POST /api/canvas-llm（支持多模态：图片帧、视频关键帧）  
● 对话页：POST /api/chat 或 /api/chat/stream
4.6 支持的 Provider 协议分发逻辑
后端 generate_ai_image() 按 provider 路由：
modelscope  → ModelScope 异步任务 API
jimeng      → 即梦 CLI subprocess
runninghub  → RunningHub OpenAPI
gemini      → Gemini 原生格式
volcengine  → 火山 Ark 签名请求
apimart     → 异步提交 + 轮询
openai      → /v1/images/generations 或 /v1/images/edits

六、商用化改造建议
若目标是独立商用产品（非个人/公司内部使用），建议从架构、合规、产品三个维度系统改造：
6.1 合规与授权（首要）
● 当前项目禁止未经授权的商业封装；商用须联系原作者取得授权  
● 若不能授权，应参考产品思路自研，避免直接 fork 闭源  
● 上游 API（APIMart、ModelScope 等）需单独确认 ToS 与转售条款
6.2 架构重构（当前最大短板）
现状	商用建议
单文件 main.py 1.48 万行	拆分为 routes/、services/、providers/、models/ 模块
JSON 文件存储	PostgreSQL（元数据）+ S3/OSS（媒体）+ Redis（缓存/队列）
内存任务 CANVAS_TASKS	Celery/RQ + 持久化任务表，服务重启不丢任务
API Key 存 .env 明文	KMS/Vault 加密，按租户隔离，禁止前端暴露
无用户体系	JWT/OAuth2 + RBAC + 多租户
CORS *	限定域名 + CSRF 防护
无鉴权	所有 /api/* 加认证中间件
6.3 产品能力增强
1. 计费与配额：按用户/团队计量 Token、生图次数、存储空间  
2. 任务中心：统一异步任务 UI，支持取消、重试、优先级队列  
3. 协作：画布实时协同（CRDT/Yjs），版本历史、评论  
4. 工作流市场：ComfyUI/RunningHub 工作流模板化、付费分发  
5. 审计日志：谁调用了哪个模型、消耗多少、生成了什么  
6. 内容安全：提示词/图片审核、水印、版权追踪
6.4 前端工程化
现状	建议
多 HTML + 原生 JS	React/Vue + TypeScript + Vite
Tailwind CDN	构建时 Tailwind，减小体积、支持 tree-shaking
无组件复用	画布引擎抽成独立 npm 包
i18n 手写 JSON	i18next + 自动化提取
6.5 运维与部署
开发: Docker Compose (api + worker + redis + postgres + minio)
生产: K8s + HPA 按 GPU/API 队列扩缩
监控: Prometheus + Grafana + Sentry
CI/CD: GitHub Actions 跑 lint/test/build
● ComfyUI 实例池化：独立 GPU 集群，与 Web 层解耦  
● 静态资源 CDN 加速  
● 健康检查 /health、优雅关停
6.6 测试与质量
当前几乎无自动化测试。商用至少需要：
● Provider 适配层单元测试（Mock 上游）  
● 画布 CRUD 集成测试  
● E2E（Playwright）：创建画布 → 生图 → 保存  
● 负载测试：并发 100 生图任务
6.7 可优先落地的 MVP 改造路径
若资源有限，建议按此顺序：
1. 模块化拆分 main.py（1–2 周）  
2. PostgreSQL + 对象存储 替换 JSON（1 周）  
3. 用户登录 + API Key 加密（1 周）  
4. Redis 任务队列 替换内存任务（1 周）  
5. React 重写画布核心页（2–4 周）  
6. 计费对接（Stripe/支付宝，视市场而定）

七、总结
Infinite Canvas 是一个功能丰富的本地 AI 创作聚合器：用 FastAPI 统一封装 7+ 种 AI 后端，用节点画布串联生图/视频/LLM/ComfyUI 工作流。技术选型偏轻量、易部署（便携 Python + 单文件后端 + 无构建前端），适合个人创作者快速上手。
其局限也明显：单体巨文件、无数据库、无用户体系、任务不持久、密钥明文、无商用授权。若要做类似商用产品，核心价值在于「多 Provider 统一编排 + 可视化工作流」的产品思路，但需要在合规授权、架构拆分、多租户、持久化队列、安全与计费上做系统性重建，而不是简单改 UI 或换 Logo。
如需针对某一模块（例如 ComfyUI 集成、APIMart 异步协议、画布数据格式）做更深入的源码级拆解，可以指定模块继续分析。
