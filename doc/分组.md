本文档专门说明**智能画布**（`smart-canvas.html` / `static/js/smart-canvas.js`）中的 **智能分组**（`type: 'smart-group'`）——从创建、入组，到顶部浮动工具栏五项功能的完整流程与源码对应关系。

## 二、分组是什么：数据模型

智能分组与普通「多图节点」不同，采用**双通道存储**：

| 字段 | 类型 | 含义 |
|------|------|------|
| `type` | `'smart-group'` | 节点类型标识 |
| `x, y, w, h` | number | 分组框位置与尺寸 |
| `title` | string | 默认「智能分组」 |
| `images` | `Image[]` | **已吸收**的图片（原图片节点被删除，媒体存在这里） |
| `items` | `string[]` | **仍留在画布上**的成员节点 id（`smart-prompt` / `smart-loop`） |
| `_memberZoom` | number? | 分组缩放倍率（默认 1） |

关键判断函数：


## 五、整理排列

### 5.1 功能说明

把分组内散乱的成员重新排成整齐网格，并**收敛分组框尺寸**刚好包住内容。适用于：

- 刚 `Ctrl+G` 成组之后
- 拖入新节点之后
- 手动拖乱了成员位置之后

成功后 toast：**「已整理分组」**。

### 5.2 用户操作

1. 选中智能分组
2. 点击工具栏 **整理排列**（`layout-grid` 图标）

### 5.3 两种整理模式

函数：`arrangeSmartGroupMembers(group, options={})`

**模式 A：有缩略图**（`group.images` 或成员图片存在）

- 用 `smartGroupThumbLayout()` 算列数、缩略图大小
- 提示词/循环（compact members）排在图片网格之后，统一缩为方格（默认 thumb 边长）
- 更新 `group.w` / `group.h`

**模式 B：仅有提示词/循环成员、无吸收图片**

- 成员按当前位置排序（先 y 后 x，容差 24px）
- 图片成员清除 `w/h` 恢复自然尺寸（修复反复拖入拖出越缩越小的问题）
- 列数 ≈ √成员数，单元格内居中
- 分组框 = 所有成员包围盒 + 内边距（pad 18、gap 16、header 44）

核心常量：

```1315:1317:static/js/smart-canvas.js
const SMART_GROUP_ARRANGE_PADDING = 18;
const SMART_GROUP_ARRANGE_GAP = 16;
const SMART_GROUP_ARRANGE_HEADER = 44;
```

### 5.4 调用链

```
点击「整理排列」
  → runSmartGroupToolbarAction(id, 'arrange')
  → arrangeSmartGroupMembers(group)   // 默认 pushUndo
  → render() + scheduleSave() + toast('已整理分组')
```

自动整理（成组/拖入时 `skipUndo: true`）：

- `groupSelectedNodes()` 末尾
- `addDraggedNodesToSmartGroup()` 末尾
- `addCreatedNodeToMenuGroup()` 末尾

## 六、预览

### 6.1 功能说明

在全屏图片编辑器（`imageEditModal`）中以 **preview 模式**浏览分组内图片。

## 七、宫格拼接

### 7.1 功能说明

把分组内**多张图片**拼成**一张大图**（join 模式），输出为新的图片节点，标题 `Grid Join`，出现在分组右侧。

与单图的「宫格切分」（split）相反：

| 模式 | `gridOperationMode` | 作用 |
|------|---------------------|------|
| 宫格切分 | `split` | 一张图切成多张 |
| **宫格拼接** | `join` | 多张图合成一张 |

分组工具栏固定走 **join**。

### 7.2 前置条件

- `group.images` 至少 **2** 张有 url 的图片
- 工具栏按钮 `enabled: imageCount > 1`

### 7.3 用户操作流程

1. 选中分组 → 点击 **宫格拼接**
2. 打开图片编辑器，自动进入 **grid + join** 模式
3. 在编辑器中可调整：行列预设、间距（gap）、拖拽调换顺序、输出尺寸（默认长边 2048）
4. 点击 **「输出拼接」**（`applyImageEdit` → `applyImageGridJoin`）
5. 生成新节点，toast「已输出拼接图片」

## 九、解散分组

### 9.1 功能说明

拆除分组容器，恢复可独立编辑的节点：

| 内容 | 解散后 |
|------|--------|
| `group.images` | 每张图 → 新建 `smart-image` 节点，按原网格位置平铺 |
| `items` 内提示词/循环 | **保留原位**，不删除 |
| 分组节点 | 删除 |
| 以分组为端点的连线 | 删除 |
| 其他节点 `inputNodeIds` | 移除该分组 id |

|------|------|
| 工具栏 | **解散分组**（`ungroup` 图标） |

### 9.3 核心实现

```14553:14591:static/js/smart-canvas.js
function ungroupNode(groupId){
    const group = nodes.find(n => n.id === groupId);
    if(isSmartGroupNode(group)){
        pushUndo();
        const memberIds = smartGroupMembers(group).map(m => m.id);
        const groupImages = (group.images || []).filter(img => img?.url);
        // 按 imageLayout 网格计算每张图的世界坐标
        created = groupImages.map((img, index) => ({
            id: uid('smart'), type:'smart-image', x, y, w, h,
            images:[stripImageGenerationMeta({...img})], ...
        }));
        nodes = nodes.filter(n => n.id !== groupId);
        nodes.push(...created);
        canvas.connections = connections.filter(c => c.from !== groupId && c.to !== groupId);
        // 清理 inputNodeIds、其他分组的 items
        selectedIds = [...created.map(n => n.id), ...memberIds];
        render(); scheduleSave();
        return true;
    }
}
```

解散后：若产生多个节点则多选；仅一个则单选。

## 十二、常见问题

**Q：为什么「整理排列」对只有提示词的分组也有效？**  
A：`hasContent` 包含 `smartGroupMembers(node).length > 0`，无图时走成员网格整理分支。

**Q：宫格拼接和单图「宫格切分」有何不同？**  
A：分组工具栏强制 `gridOperationMode = 'join'`；单图节点菜单在仅 1 张图时显示「宫格切分」（split）。

**Q：预览能否浏览 `items` 里成员节点上的图？**  
A：工具栏预览只看 `group.images`。若成员图片节点未吸收进组，应通过 `smartGroupImageRefs` 路径（双击成员图 → `openImagePreviewSmart`）。

**Q：解散后连线会恢复吗？**  
A：不会自动恢复到各拆出的图片节点；以分组为端点的连线会被删除。入组时 `rerouteSmartConnections` 把线改到分组，解组时需手动重连。
